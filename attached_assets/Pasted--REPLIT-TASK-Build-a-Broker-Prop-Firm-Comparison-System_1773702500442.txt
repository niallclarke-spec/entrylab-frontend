# REPLIT TASK: Build a Broker & Prop Firm Comparison System for EntryLab.io

---

## CRITICAL RULES — READ BEFORE DOING ANYTHING

1. **EVERY comparison page MUST be created as a DRAFT. Never publish automatically.** The admin will manually publish pages via an admin dashboard. No exceptions.
2. **Drafts are NOT visible to the public.** They do not appear on hub pages, in the sitemap, in navigation, or in internal links. Only published pages are visible.
3. **When a draft is published**, THEN and ONLY THEN: add it to the sitemap, show it on hub pages, include it in internal linking sections, and make it crawlable.
4. **URLs are always alphabetical.** `avatrade-vs-ic-markets` — never the reverse. If someone visits the reverse URL, 301 redirect to the alphabetical version.
5. **This system covers TWO entity types:** Brokers AND Prop Firms. Both follow the same architecture but have different comparison categories and data fields. Build both.

---

## WHAT YOU ARE BUILDING

A system that:
- Auto-generates a comparison page (as a draft) for every possible pair of brokers and every possible pair of prop firms
- Lets the admin publish drafts on their own schedule via an admin dashboard
- Auto-generates new drafts whenever a new broker or prop firm is added
- Handles the full lifecycle: draft → published → updated → archived
- Includes hub pages, navigation, internal linking, schema markup, and FAQ sections

## SECTION 1: DATABASE

### 1A. Comparison Record Schema

Create a `comparisons` table/collection with these fields:

| Field | Type | Description |
|-------|------|-------------|
| id | string/uuid | Unique ID |
| entity_type | enum | `broker` or `prop_firm` |
| entity_a_id | foreign key | References the first broker/prop firm |
| entity_b_id | foreign key | References the second broker/prop firm (alphabetically after A) |
| slug | string | Auto-generated: `{entity-a-slug}-vs-{entity-b-slug}` |
| status | enum | `draft`, `published`, `updated`, `archived` |
| category_winners | JSON | Object storing the winner of each of the 7 categories |
| overall_winner_id | foreign key | Which entity won the most categories |
| overall_score | string | e.g. "4-3" or "5-2" |
| created_at | datetime | When the draft was generated |
| published_at | datetime | When the admin published it (null if draft) |
| updated_at | datetime | Last time data was refreshed |

**IMPORTANT:** `entity_a` must ALWAYS be alphabetically before `entity_b` by slug. Enforce this at creation time.

### 1B. Additional Fields on Broker Schema

If these fields do not already exist on the broker model, add them:

- `spreads` — string (e.g. "From 0.0 pips")
- `commission` — string (e.g. "$7/lot" or "Commission-free")
- `year_founded` — string (e.g. "2019")
- `headquarters` — string (e.g. "United Kingdom")
- `account_types` — array of strings (e.g. ["Standard", "ECN", "VIP"])
- `overall_rating` — number (e.g. 7.5, out of 10)

### 1C. Additional Fields on Prop Firm Schema

If these fields do not already exist on the prop firm model, add them:

- `challenge_types` — string (e.g. "1-Step, 2-Step, Instant Funding")
- `profit_split` — string (e.g. "Up to 90%")
- `max_funding` — string (e.g. "$200,000")
- `challenge_price` — string (e.g. "From $49")
- `profit_target` — string (e.g. "8% Phase 1, 5% Phase 2")
- `daily_drawdown` — string (e.g. "5%")
- `max_drawdown` — string (e.g. "10%")
- `payout_frequency` — string (e.g. "Bi-weekly")

## SECTION 4: COMPARISON CATEGORIES & WINNER LOGIC

### 4A. Broker vs Broker — 7 Categories

**Category 1: Regulation & Safety**
- Compare: regulation field values
- Winner logic: Tier 1 (FCA, ASIC, CySEC, BaFin, MAS, FINMA, FMA NZ, JFSA) beats Tier 2 (FSA Seychelles, FSCA, DFSA, FSC Mauritius) beats Offshore/Unregulated. More regulators beats fewer at the same tier.

**Category 2: Trading Costs**
- Compare: spreads and commission fields
- Winner logic: Lower EUR/USD spread wins. If equal, lower commission wins.

**Category 3: Platforms & Tools**
- Compare: trading_platforms field
- Winner logic: Count of unique platforms. More = winner. Proprietary platform = bonus point. Copy trading = bonus point.

**Category 4: Account Types & Minimum Deposit**
- Compare: account_types array and minimum_deposit field
- Winner logic: Lower minimum deposit wins. More account types = tiebreaker.

**Category 5: Deposit & Withdrawal**
- Compare: deposit_methods and minimum_withdrawal fields
- Winner logic: More deposit methods wins. Crypto support = bonus. Lower/no withdrawal fees = bonus.

**Category 6: Leverage & Instruments**
- Compare: max_leverage field
- Winner logic: Higher max leverage wins. More instrument types = tiebreaker.

**Category 7: Customer Support & Reputation**
- Compare: available data on support channels, Trustpilot rating if stored
- Winner logic: Live chat beats email only. 24/7 beats business hours. Higher rating wins.

### 4B. Prop Firm vs Prop Firm — 7 Categories

**Category 1: Challenge Structure**
- Compare: challenge_types, profit_target fields
- Winner logic: More challenge options wins. Lower profit targets = more trader-friendly = bonus.

**Category 2: Pricing**
- Compare: challenge_price field
- Winner logic: Lower challenge fee for equivalent account size wins.

**Category 3: Profit Split & Payouts**
- Compare: profit_split, payout_frequency fields
- Winner logic: Higher profit split percentage wins. More frequent payouts = tiebreaker.

**Category 4: Drawdown Rules**
- Compare: daily_drawdown, max_drawdown fields
- Winner logic: Higher (more lenient) drawdown limits win. Static drawdown beats trailing drawdown.

**Category 5: Trading Platforms**
- Compare: trading_platforms field
- Winner logic: Same as broker Category 3.

**Category 6: Scaling & Max Funding**
- Compare: max_funding field
- Winner logic: Higher max funding wins.

**Category 7: Reputation & Trust**
- Compare: Trustpilot rating, year_founded, payout proof if available
- Winner logic: Higher rating wins. Longer in business = tiebreaker.

## SECTION 9: AUTO-GENERATION TRIGGERS

### 9A. New Broker/Prop Firm Added
When a new broker or prop firm record is created and saved:
1. Loop through ALL existing entities of the same type
2. Create a draft comparison for each pair involving the new entity
3. Calculate winners and scores for each draft
4. Do NOT publish, do NOT add to sitemap or hub pages

### 9B. Broker/Prop Firm Data Updated
When any field on a broker or prop firm is edited:
1. Find all comparison records (any status) involving that entity
2. Recalculate the comparison data, winners, and scores
3. If the comparison is published, set status to `updated` and update `updated_at`
4. If the comparison is a draft, just update the data silently

### 9C. Broker/Prop Firm Marked as Closed/Inactive
When an entity is marked as closed or inactive:
1. Set all its comparison records to status `archived`
2. Published comparison pages should show a banner: "{Entity} is no longer operating. View alternatives →"
3. Remove archived pages from the sitemap
